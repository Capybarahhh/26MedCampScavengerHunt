# MNEMO City Scavenger Hunt ‚Äî Architecture

A Vite + React 18 single-page game for the 2026 med-camp city scavenger hunt.
This replaces the original single-file DC artifact (kept in `.dev/legacy/` for
reference). The rewrite is behavior- and visual-parity with the legacy version.

## Source layers ‚Äî content / mechanics / presentation

**`src/data/` ‚Äî content only, zero logic.** Editing story text, puzzle
answers, map labels or food prices happens here and nowhere else.

| File | Contents |
|---|---|
| `stages.js` | Every stage's beats: story text, puzzles, **answers** (search `answer:`) |
| `intro.js` | Opening narration pages |
| `stageMeta.js` | Map node names/coordinates + node icon shapes |
| `food.js` | Delivery mini-game menu items and prices |
| `typeChart.js` | Pok√©mon type-effectiveness chart (duel rules beat) |
| `cipher.js` | Vigen√®re table cells (final MEMORY LOCK puzzle) |
| `clsStyle.js` | Text-run colors per narrative class (narr/quote/rule/‚Ä¶) |

**`src/hooks/` ‚Äî game mechanics.**
`useTypewriter` (char-by-char text with tap-to-skip), `useScale` (fits the
fixed 700√ó800 canvas to the viewport), `useFoodGame` (the whole delivery
mini-game state machine and its timers).

**`src/components/` ‚Äî presentation.**
- `screens/` ‚Äî one file per full screen: Entry (room code), Intro, Map,
  Stage, Backpack, Assembly (drag-and-drop jigsaw), Endings.
- `beats/` ‚Äî the units inside a stage: StoryBeat, PuzzleBeat, ColorPickBeat,
  FoodGameBeat, FragmentBeat. `StageScreen` dispatches on `beat.type`.
- `ui/` ‚Äî shared pieces: `TerminalPanel` (the CRT panel frame used
  everywhere), `AnswerTerminal` (the status-tinted answer-input frame shared
  by puzzle and color-pick beats), `SegText` (styled narrative text under a
  typewriter budget), `NavButtons` (prev/next row).
- `CityBackdrop.jsx` ‚Äî ambient city background (buildings, rain, particles,
  HUD corners) behind every screen.

**`src/lib/` ‚Äî pure helpers.**
`css.js` (CSS-string ‚Üí style object + `mix()` alpha-fade helper),
`pieces.js` (jigsaw geometry + fragment SVG), `text.js` (seg/paragraph
utils), `map.js` (route curve), `backdrop.js` (ambience generators),
`progress.js` (localStorage save/load), `track.js` (analytics, below).

## Styling & theming (art-style testing)

The palette lives in **`src/styles/tokens.css`** as CSS custom properties on
`:root` ‚Äî components never hardcode theme colors; they reference
`var(--teal)`, `var(--purple-btn)`, `rgba(var(--gold-rgb), .5)`, etc.
(`src/styles/global.css` holds only keyframes and :active/:focus classes.)

- **To try a new art style**: copy a `[data-theme="‚Ä¶"]` block in tokens.css,
  rename it, override the tokens you care about (unset ones inherit the
  default), then open the game with `?theme=yourname`. The choice persists in
  localStorage; `?theme=default` switches back. Two demo themes ship as
  proof: `?theme=ember` (amber/rust) and `?theme=ice` (glacial blue).
- **Alpha fades of a token** go through `mix(color, pct)` from `lib/css.js`
  (CSS `color-mix`), never string-concatenated hex suffixes ‚Äî those break
  with `var()` references.
- **Deliberately NOT tokenized** (content, not theme): per-panel accents in
  `data/stages.js` (e.g. the G stage's `#8fe23c`), colorpick swatches, and
  the fragment paper golds in `lib/pieces.js`.
- `.dev/smoke.mjs` (`node .dev/smoke.mjs` after `npm run build`) mounts the
  built bundle in happy-dom and walks map ‚Üí stage as a crash check.

**`src/App.jsx` ‚Äî the controller.** Owns the screen router and all
cross-screen progress (unlocked stage index, collected fragments, current
stage/beat). Everything beat-local (typewriters, puzzle inputs, mini-game
clocks) lives inside the screen components and resets on remount.

## Game flow

entry ‚Üí intro ‚Üí map ‚Üí stage (beats: story/puzzle/foodgame/fragment) ‚Üí ‚Ä¶ ‚Üí
END stage ‚Üí assembly ‚Üí final puzzle ‚Üí ending choice ‚Üí ending1/ending2.

- Room code `000000` enables **creator mode**: all map nodes unlocked.
- Progress persists in localStorage key `mnemo_progress` (same shape as the
  legacy app, so old saves still load). Session id in `mnemo_session`.

## Interaction tracking

`src/lib/track.js` sends fire-and-forget POSTs to `/api/track` (override with
`VITE_TRACK_ENDPOINT`). Failures are silent and never block gameplay.

Events: `session_start`, `room_confirmed`, `stage_start`, `puzzle_attempt`
(includes wrong answers), `fragment_collected`, `foodgame_end`,
`assembly_complete`, `ending_choice`, `reset`. All carry `sessionId`,
`roomCode` (doubles as the team id) and `ts`.

`api/track.js` forwards events to the `TRACK_WEBHOOK_URL` env var if set
(e.g. a Google Apps Script that appends to a Sheet); otherwise it just logs.

## Workflows

- **Develop**: `npm run dev` ‚Üí http://localhost:5173/ (hot-reloads on save).
- **Build check**: `npm run build`.
- **Deploy**: push to `main` ‚Üí `.github/workflows/deploy.yml` builds and
  publishes to GitHub Pages. Full setup (Google Sheet backend, team codes,
  Pages switch): **docs/DEPLOY.md**. Team room codes live in
  `src/data/teams.js`.
- **Verify a change**: walk the game with room code `000000`; compare against
  `.dev/legacy/MNEMO.dc.html` (open directly in a browser) if visuals drift.
